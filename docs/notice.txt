  UTB MINUTE — Setup and operations
================================================================================

APPLICATION OVERVIEW
   - Two Blazor clients: AdminClient (Admin: foods + menu), CanteenClient (Student + Cook).
   - One WebApi: foods, menu, orders, real-time SSE at GET /sse/orders.
   - Keycloak for login and roles: Admin, Student, Cook.
   - SQL Server and DbManager (POST /db/reset). WebApi seeds DB on startup (EnsureCreated + MinuteDbSeed).

-------------------------------------------------------------------------------
1. FIRST-TIME START
-------------------------------------------------------------------------------
   - Start Docker Desktop.
   - Set UTB.Minute.AppHost as startup project; run it.
   - Aspire Dashboard opens (e.g. http://localhost:15120). Wait until all resources are running.
   - Keycloak realm "minute" is imported from AppHost/Realms (e.g. minute-realm.json) on first start.
   - If the realm or minute-blazor client is missing, the Keycloak volume may already exist from a previous run.
     Realm import runs only when the realm does not exist. Fix: stop AppHost → docker volume ls (find keycloak volume)
     → docker volume rm <name> → restart AppHost.
   - Keycloak Admin: use the URL from Aspire Dashboard (Resources → keycloak). Login admin/admin (or value from
     AppHost parameters). Switch realm to "minute" (dropdown top-left). Create users and assign realm roles
     Admin, Cook, or Student (e.g. admin/admin, cook/cook, student/student).
   - Database: WebApi runs EnsureCreatedAsync and MinuteDbSeed on startup. To reset: POST http://localhost:<port>/db/reset
     (DbManager port is in Aspire Dashboard → Resources → dbmanager → endpoint).

-------------------------------------------------------------------------------
2. ROLES AND APPS
-------------------------------------------------------------------------------
   - Admin: AdminClient only. Food Management (/), Menu Management (/menu), Access Token (/token). Cannot place orders.
   - Student: CanteenClient. Today's Menu (/), My Orders (/my-orders). Can place orders; My Orders updates in real time (SSE).
   - Cook: CanteenClient. Kitchen (/orders) only. Today's Menu and My Orders are hidden in nav. Orders list updates in real time (SSE).
   - NavMenu in each app uses AuthorizeView / IsInRole to show links per role.

-------------------------------------------------------------------------------
3. REAL-TIME (SSE)
-------------------------------------------------------------------------------
   - WebApi: GET /sse/orders (AllowAnonymous). Sends "order-created" and "order-updated" when orders are created or status changes.
   - CanteenClient: MyOrders.razor and Orders.razor subscribe to that stream (client "api-sse") and refetch their list on any event,
     so the UI updates without refresh. Subscription is cancelled when the user leaves the page (IDisposable).

-------------------------------------------------------------------------------
4. KEYCLOAK EXPORT / IMPORT (use on new machine or backup)
-------------------------------------------------------------------------------
   Export (run where Keycloak volume already exists; replace "keycloak" with your volume name from docker volume ls):
     docker run --rm `
       -v "C:\temp\kc-export:/opt/keycloak/data/export" `
       -v "keycloak:/opt/keycloak/data" `
       quay.io/keycloak/keycloak:26.4 `
       export --dir /opt/keycloak/data/export

   Transfer the folder C:\temp\kc-export to the new machine (e.g. zip and copy).

   Import on new machine (create volume, then import from the copied folder):
     docker volume create keycloak-restored
     docker run --rm `
       -v "C:\temp\kc-export:/opt/keycloak/data/import" `
       -v "keycloak-restored:/opt/keycloak/data" `
       quay.io/keycloak/keycloak:26.4 `
       import --dir /opt/keycloak/data/import

   To use with Aspire: either rename the volume to match the one Aspire creates for Keycloak (check after first run with docker volume ls),
   or copy the imported data into that volume. Full step-by-step: docs/DOCKER-VOLUMES.md.

-------------------------------------------------------------------------------
5. TROUBLESHOOTING
-------------------------------------------------------------------------------
   Logout "Missing parameters: id_token_hint"
     - Keycloak 26+ may require valid post-logout redirect URIs on the minute-blazor client. Add your app roots
       (e.g. http://localhost:5024/, http://localhost:5036/) in Keycloak → Clients → minute-blazor → Valid post logout redirect URIs.

   Redirect URI errors (invalid_grant, Incorrect redirect_uri after login/logout)
     - Ensure app redirect URIs match Keycloak client settings. If you changed ports or realm config, delete the Keycloak
       volume and restart AppHost so the realm is re-imported; then recreate users in Keycloak Admin.

   Roles not working (401 on API or wrong menu)
     - In Keycloak, ensure realm roles Admin/Student/Cook exist and are assigned to users. Ensure the client (minute-blazor)
       has a scope that adds realm roles to the token (e.g. claim "role"), and that WebApi TokenValidationParameters
       use the same RoleClaimType.
